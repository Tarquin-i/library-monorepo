# 图书管理系统需求文档

## 一、项目概述

### 1.1 项目目标
开发一套图书管理系统，实现图书馆日常书籍管理、借阅流程的数字化，核心目标：
- 实现多角色精细化权限控制
- 简化书籍录入流程，支持单个/批量录入
- 规范书籍借阅、归还、续借、逾期处理流程
- 支持书籍、借阅记录的快速查询与导出

### 1.2 核心模块
1. **权限控制模块**：多角色权限管理
2. **书籍录入模块**：书籍信息录入与管理
3. **书籍借阅模块**：借阅、归还、续借、记录管理

---

## 二、权限控制模块

### 2.1 角色定义

#### 2.1.1 管理员（超级管理员）
- **权限范围**：系统最高权限
- **核心操作**：
  - 系统管理：所有系统功能操作权限
  - 用户管理：新增、删除、修改用户角色
  - 书籍管理：书籍全流程管理（录入、修改、删除、借阅、归还）

#### 2.1.2 图书管理员
- **权限范围**：图书日常业务操作权限
- **核心操作**：
  - 书籍录入、修改、删除
  - 书籍借阅、归还、续借办理
  - 用户操作权限
  - 书籍查询与统计

#### 2.1.3 读者
- **权限范围**：个人业务操作权限
- **核心操作**：
  - 查询书籍信息、馆藏状态
  - 申请借阅、归还图书
  - 查看个人借阅记录
  - 修改个人基础信息

### 2.2 权限管控规则
1. **角色唯一性**：每个用户只能拥有一个角色
2. **角色分配**：管理员可以分配和修改用户角色
3. **登录验证**：所有用户需通过账号密码登录系统
4. **操作拦截**：系统自动识别用户角色，对越权操作进行拦截

## 三、书籍录入模块

### 3.1 单个书籍录入

#### 3.1.1 录入权限
- 仅管理员和图书管理员可进行书籍录入操作

#### 3.1.2 必填信息
- ISBN 编号
- 书名
- 作者
- 出版社
- 出版日期
- 书籍分类
- 价格
- 馆藏数量
- 可借阅数量
- 书籍位置
- 书籍简介
- 书籍封面图片（可选）

#### 3.1.3 录入校验规则
1. **ISBN 唯一性校验**：
   - 如果 ISBN 已存在，不允许重复录入
   - 需要增加数量时，修改已有书籍的馆藏数量
2. **数量校验**：
   - 可借阅数量 ≤ 馆藏数量
   - 馆藏数量 ≥ 0
3. **书籍状态**：
   - 录入成功后，如有可借阅数量，状态自动设为"可借"

### 3.2 批量书籍录入
1. **Excel 模板导入**：
   - 支持上传 Excel 模板批量导入书籍信息
   - 系统提供标准 Excel 模板下载
   - 模板包含所有必填与可选字段
2. **数据校验**：
   - 系统自动校验导入数据
   - 对格式错误、信息缺失、重复 ISBN 的条目进行标记
   - 生成错误报告，管理员可修改后重新导入

### 3.3 书籍信息管理

#### 3.3.1 书籍信息修改
- 支持修改已录入书籍的非核心信息（作者、出版社、馆藏数量、位置等）
- **ISBN 编号一旦录入不可修改**

#### 3.3.2 书籍删除
- 仅可删除无借阅记录的书籍（无在借、无历史借阅）
- 删除后数据从系统中移除

#### 3.3.3 书籍状态管理
- 系统自动更新书籍状态，分为：
  - **可借**：在馆且有可借阅数量
  - **已借**：所有副本均已借出
  - **遗失**：书籍遗失，管理员手动标记
  - **报废**：书籍报废，管理员手动标记

## 四、书籍借阅模块

### 4.1 借阅申请

#### 4.1.1 申请权限
- 读者登录系统后，可对"可借"状态的书籍发起借阅申请

#### 4.1.2 借阅限制
1. **数量限制**：单个读者最多可借阅 5 本书籍
2. **期限限制**：借阅期限默认为 30 天
3. **逾期限制**：如有逾期未归还的书籍，不能发起新的借阅申请

#### 4.1.3 申请流程
- 读者选择需借阅书籍，提交申请
- 系统生成借阅申请单
- 管理员审核并办理借阅

### 4.2 借阅办理

#### 4.2.1 办理权限
- 仅管理员和图书管理员可办理借阅

#### 4.2.2 办理流程
1. 管理员查看借阅申请
2. 核实读者信息与书籍状态
3. 确认借阅后：
   - 系统自动更新书籍状态为"已借出"
   - 减少可借阅数量
   - 生成正式借阅记录
   - 记录借阅时间、应归还时间（借阅时间 + 30 天）

### 4.3 书籍归还

#### 4.3.1 归还方式
- 读者可线上提交归还申请
- 或直接到馆由管理员办理归还

#### 4.3.2 归还流程
1. 管理员核实借阅记录
2. 确认书籍完好后，办理归还
3. 归还成功后：
   - 系统自动更新书籍状态为"可借"
   - 增加可借阅数量
   - 记录实际归还时间
   - 关闭借阅记录

#### 4.3.3 逾期处理
- 如实际归还时间超过应归还时间，系统自动计算逾期天数
- 生成逾期记录

### 4.4 书籍续借

#### 4.4.1 续借权限
- 仅读者可对未逾期的书籍申请续借

#### 4.4.2 续借规则
1. 每本图书仅可续借 1 次
2. 续借期限为 15 天
3. 续借后系统更新应归还时间

#### 4.4.3 办理流程
- 读者提交续借申请
- 管理员审核通过后，系统更新借阅记录
- 无需归还实物书籍

### 4.5 借阅记录管理

#### 4.5.1 读者端
- 可查看个人所有借阅记录
- 包含：书籍信息、借阅时间、应归还时间、实际归还时间、逾期状态

#### 4.5.2 管理员端
- 可查询所有读者的借阅记录
- 支持按以下条件筛选：
  - 书籍名称
  - 读者姓名
  - 借阅时间
  - 逾期状态
- 支持导出借阅记录报表

## 五、数据需求

### 5.1 核心数据实体

#### 5.1.1 用户信息表
- 用户 ID
- 账号
- 密码（加密存储）
- 姓名
- 角色（管理员/图书管理员/读者）
- 联系方式
- 注册时间
- 账号状态

#### 5.1.2 书籍信息表
- 书籍 ID
- ISBN 编号（唯一）
- 书名
- 作者
- 出版社
- 出版日期
- 书籍分类
- 价格
- 馆藏数量
- 可借阅数量
- 书籍位置
- 书籍简介
- 书籍封面图片
- 书籍状态（可借/已借/遗失/报废）
- 录入时间

#### 5.1.3 借阅记录表
- 借阅 ID
- 用户 ID
- 书籍 ID
- 借阅时间
- 应归还时间
- 实际归还时间
- 逾期天数
- 归还状态（在借/已归还）
- 续借次数
- 操作管理员 ID

### 5.2 数据流转规则
1. **书籍录入**：数据同步至书籍信息表，状态默认"可借"
2. **借阅办理**：生成借阅记录，同步更新书籍可借阅数量与状态
3. **归还办理**：更新借阅记录实际归还时间，同步恢复书籍可借阅数量与状态
4. **续借办理**：更新借阅记录应归还时间，增加续借次数

---

## 六、非功能需求

### 6.1 性能需求
- 系统响应速度：页面加载时间 ≤ 3 秒
- 操作响应时间：书籍查询、录入、借阅办理等操作响应时间 ≤ 2 秒

### 6.2 安全性需求
- 用户密码加密存储
- 系统记录关键操作日志（录入、借阅、权限修改等）
- 支持内网部署

### 6.3 易用性需求
- 界面设计简洁清晰，操作流程直观
- 系统操作提示明确，错误信息清晰易懂

---

## 七、验收标准

1. **功能验收**：所有权限控制、书籍录入、书籍借阅功能均实现，无功能缺失
2. **权限验收**：不同角色用户权限清晰，越权操作可成功拦截
3. **数据验收**：数据录入、查询、流转准确无误，无数据丢失、错误情况
